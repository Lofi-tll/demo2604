图书管理系统

项目简介

本项目是基于SpringBoot3开发的前后端分离图书管理系统，采用 MVC 三层架构 + 多模块设计，实现学生、管理员双角色权限管控，完成图书分类管理、图书借阅 / 归还、借阅记录追踪、超期标记等核心图书管理业务，满足校园 / 小型图书馆日常图书管理需求，代码结构清晰、业务逻辑完善，适配企业级开发规范。
技术栈

后端核心

基础框架：SpringBoot 3.2.5

开发语言：Java 17

数据库：MySQL 9.5

ORM 框架：MyBatis-Plus（MP）

安全框架：Spring Security
令牌认证：JWT

日志框架：SLF4J

接口文档：Knife4j

其他：注解式事务、全局异常处理、自定义 JWT 工具类、过滤器

未集成技术

Redis（待后续扩展优化）

项目架构

采用MVC 三层架构：Controller 层（接口请求处理）、Service 层（业务逻辑实现）、Dao 层（数据持久化）

采用多模块设计：拆分通用工具模块、核心业务模块、配置模块等，降低代码耦合度，便于维护扩展

权限架构：SpringSecurity+JWT 实现双角色权限拦截，不同角色访问对应业务接口

核心功能模块

1. 学生角色功能
   
图书查询：支持图书分类筛选、图书信息详情查询

图书借阅：申请借阅可借阅状态图书，记录借阅时间

图书归还：归还已借阅图书，自动校验借阅时长

借阅记录：查看个人所有借阅、归还、超期记录

2. 管理员角色功能

图书管理：新增图书、编辑图书信息、图书上下架操作

分类管理：支持二级分类管理，父分类仅用于归类，不可存放图书，二级分类绑定图书

借阅管理：查看全量用户借阅记录、查询单个用户借阅状态

超期管理：自动标记超期未归还图书，同步更新借阅状态

3. 公共核心业务

图书分类查询 + 图书信息分页查询

图书借阅资格校验 + 借阅流程处理

图书归还校验 + 超期状态自动判定

多维度借阅记录查询

数据库设计核心要点

图书分类：父子分类结构，父分类无图书绑定，二级分类关联图书

图书信息：包含分类、库存、可借阅状态、上下架状态字段

借阅记录：关联用户、图书信息，记录借阅时间、应归还时间、实际归还时间、超期状态

开发环境配置

环境要求

JDK：17 及以上

MySQL：9.5

Maven：3.6 及以上

开发工具：IntelliJ IDEA

项目启动与接口访问

启动 SpringBoot 主启动类，项目成功运行

接口文档访问：启动项目后，浏览器访问 http://localhost:端口/doc.html，打开 Knife4j 接口文档，可查看所有接口信息、在线调试接口

权限说明：不同角色接口需携带对应 JWT 令牌请求，未登录 / 无权限接口会被拦截

项目核心特性

权限管控：SpringSecurity+JWT 实现双角色细粒度权限控制，接口安全校验

事务管理：核心业务采用注解式事务，保证数据一致性

异常处理：自定义全局异常处理器，统一返回异常结果，提升接口友好性

工具封装：封装 JWT 生成、解析、校验工具类，自定义权限过滤器，简化业务代码

业务规范：严格遵循 MVC 分层、多模块开发规范，代码可读性、可扩展性强
